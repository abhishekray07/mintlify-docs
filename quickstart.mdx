## Setup Opslane

Follow these steps to set up Opslane and start improving your on-call experience.

### Prerequisites

Before you begin, make sure you have:

- Docker installed on your system
- A Slack workspace
- A Datadog account
- OpenAI API key (for AI-powered features)

### Installation

<AccordionGroup>
  <Accordion icon="github" title="Clone the repository">
    Open your terminal and run the following command to clone the Opslane repository:

    ```bash
git clone https://github.com/opslane/opslane.git
cd opslane
```
</Accordion>

  <Accordion icon="gear" title="Configure environment variables">
    1. Copy the example environment file:

    ```bash
cp .env.example .env
```

    2. Open the `.env` file in your preferred text editor.
    3. Edit the file to add your Slack, Datadog, and OpenAI API keys.
  </Accordion>

  <Accordion icon="docker" title="Build and run with Docker">
    Build and run the Opslane Docker container:

    ```bash
docker-compose up --build
```

 This command will start Opslane and make it ready for use.
  </Accordion>
</AccordionGroup>

## Configure Integrations

<AccordionGroup>
  <Accordion icon="slack" title="Set up Slack integration">
    1. Add the Opslane bot to the Slack channel where you receive alerts.
    2. Ensure the bot has necessary permissions to read and post messages.
  </Accordion>

  <Accordion icon="dog" title="Configure Datadog">
    1. Log in to your Datadog account.
    2. Navigate to the Integrations section.
    3. Set up a new webhook integration pointing to Opslane's webhook endpoint.
    4. Configure your alerts to use this new webhook.
  </Accordion>
</AccordionGroup>

## Start Using Opslane

Once you've completed the setup and configuration:

1. Opslane will automatically start analyzing incoming alerts from Datadog.
2. Check your configured Slack channel to see Opslane's insights and recommendations.
3. Interact with Opslane in Slack to get more information or take actions on alerts.
